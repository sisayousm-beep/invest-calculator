# 투자 판단 앱 — 전문가급 설계도 (v1.0)

> **문서 성격**: 본 설계도는 개발 시 옆에 두고 참조하는 기술 명세서다. 모든 수치·공식·임계값은 "근거 있는 초안"이며, 실제 운영 중 백테스트로 조정 가능하도록 **설정 파일(config)** 한 곳에 모은다.
>
> **확정된 전제** (이전 논의 기준)
> - 대상: 한국 + 미국 주식, 티커 직접 입력
> - 데이터: `yfinance` (비공식, 무료) / 일봉 기준 / 검색 시 스냅샷 1회 로드
> - 신뢰도·타이밍 %는 **통계적 확률이 아닌 규칙 기반 "신호 일치도(휴리스틱 점수)"**
> - 가격대 역산은 휴리스틱(근사) 방식
> - 즐겨찾기는 브라우저 localStorage 저장, 계정 시스템 없음
> - 화면: ① 종목 상세 화면 + ② 즐겨찾기 비교 대시보드 둘 다
> - UI: 토스증권 스타일(세련·깔끔·전문적)

## 0. 용어 정의 (오해 방지용)

| 표기 | 실제 의미 | 주의 |
|---|---|---|
| 신뢰도 (%) | 추세 판정 근거 조건들이 **몇 개나 한 방향으로 일치하는가**를 점수화 | 통계적 적중 확률 아님 |
| 타이밍 점수 (%) | 진입 신호 조건들의 충족 정도 | 미래 수익 보장 아님 |
| 종합 평가 | 추세·타이밍·지표를 가중 합산한 5등급 | 투자 권유 아님(면책 표기 필수) |

> 화면에는 직관성을 위해 %로 표기하되, 앱 내 안내 문구에 "이 수치는 과거 통계 확률이 아니라 지표 신호의 일치 정도입니다"를 명시한다.

## 2. 데이터 레이어

### 2.1 데이터 수집
- 라이브러리: `yfinance`
- 호출: `yf.Ticker(ticker).history(period="1y", interval="1d")`
- 지표 계산을 위해 **최소 1년치 일봉** 확보 (일목 선행스팬2가 52일 + 26일 시프트를 쓰므로 워밍업 구간 필요. 안전하게 1년 권장)

### 2.2 티커 처리 규칙
| 시장 | 입력 예 | 내부 변환 |
|---|---|---|
| 미국 | `AAPL` | 그대로 |
| 한국(코스피) | `005930` | `005930.KS` 자동 부착 |
| 한국(코스닥) | `247540` | `.KQ` 시도 → 실패 시 `.KS` 폴백 |

> 숫자 6자리면 한국 종목으로 간주하고 `.KS`/`.KQ`를 순차 시도. 실패 시 사용자에게 "티커를 확인해 주세요" 안내.

- yfinance는 과도 호출 시 일시 차단될 수 있음 → **동일 티커 N분 내 재요청은 캐시 반환**
- 권장: 메모리 캐시(서버 재시작 시 소멸) 또는 간단한 파일 캐시. TTL 기본 10분.

## 3. 지표 계산 (정확한 공식)

> 모든 지표는 **일봉 종가/고가/저가** 기준. 파라미터는 표준값 사용, config로 변경 가능.

### 3.1 RSI (기간 14, Wilder 방식)
```
변화량 = 종가[t] - 종가[t-1]
상승폭 = max(변화량, 0),  하락폭 = max(-변화량, 0)
평균상승 = Wilder평활(상승폭, 14)
평균하락 = Wilder평활(하락폭, 14)
RS = 평균상승 / 평균하락
RSI = 100 - (100 / (1 + RS))
```
- Wilder 평활: 첫 값은 단순평균, 이후 `이전평균*(13/14) + 당일값*(1/14)`
- 과매수 70 / 과매도 30 (config)

### 3.2 MACD (12, 26, 9)
```
MACD선   = EMA(종가, 12) - EMA(종가, 26)
시그널선 = EMA(MACD선, 9)
히스토그램(오실레이터) = MACD선 - 시그널선
```
- 골든크로스: MACD선이 시그널선을 아래→위로 돌파
- 데드크로스: 그 반대

### 3.3 일목균형표 (9, 26, 52)
```
전환선     = (최근 9일 최고가 + 최근 9일 최저가) / 2
기준선     = (최근 26일 최고가 + 최근 26일 최저가) / 2
선행스팬1  = (전환선 + 기준선) / 2          → 26일 앞으로 이동
선행스팬2  = (최근 52일 최고가 + 최근 52일 최저가) / 2  → 26일 앞으로 이동
후행스팬   = 당일 종가                       → 26일 뒤로 이동
구름대(雲) = 선행스팬1 ~ 선행스팬2 사이 영역
```
- 양운(상승 구름): 선행스팬1 > 선행스팬2
- 음운(하락 구름): 선행스팬1 < 선행스팬2

### 3.4 볼린저밴드 (20, 2σ)
```
중심선 = SMA(종가, 20)
상단   = 중심선 + 2 × 표준편차(종가, 20)
하단   = 중심선 - 2 × 표준편차(종가, 20)
%B     = (현재가 - 하단) / (상단 - 하단)        # 0=하단, 1=상단
밴드폭 = (상단 - 하단) / 중심선                  # 변동성/스퀴즈 판단
```

### 3.5 차트 배치
- **본차트(메인)**: 캔들 + 일목균형표(전환·기준·구름·후행) + 볼린저밴드(상·중·하)
- **보조차트**: RSI(0~100, 30/70 가이드라인), MACD(MACD선·시그널선·히스토그램 막대)

## 4. 추세 판별 엔진

일목균형표 + 볼린저밴드로 **기계적 판정**. 결과는 `상승추세` / `하락추세` / `횡보(박스권)` + 신뢰도(%).

### 4.1 판정에 쓰는 조건 (각 조건 = 1점)

| # | 조건 | 상승 신호 | 하락 신호 |
|---|---|---|---|
| C1 | 주가 vs 구름대 | 구름대 위 | 구름대 아래 |
| C2 | 구름 색 | 양운 | 음운 |
| C3 | 전환선 vs 기준선 | 전환선 > 기준선 | 전환선 < 기준선 |
| C4 | 후행스팬 vs 26일 전 주가 | 위 | 아래 |
| C5 | 볼린저 중심선 기울기 | 상승 | 하락 |

### 4.2 판정 로직
```
상승점수 = (상승 신호인 조건 개수)   # 0~5
하락점수 = (하락 신호인 조건 개수)   # 0~5

if 주가가 구름대 내부 또는 밴드폭 < 스퀴즈임계값:
    추세 = "횡보"
elif 상승점수 >= 3 and 상승점수 > 하락점수:
    추세 = "상승추세"
elif 하락점수 >= 3 and 하락점수 > 상승점수:
    추세 = "하락추세"
else:
    추세 = "횡보"   # 신호 충돌 시 보수적으로 횡보 처리
```

### 4.3 신뢰도(%) 산출
```
신뢰도 = (지배적 방향의 조건 점수 / 5) × 100
# 예: 상승추세인데 상승 조건 4개 충족 → 신뢰도 80%
# 횡보의 신뢰도 = (1 - 밴드폭정규화) 기반 또는 충돌도 기반 (config로 정의)
```
- **횡보 신뢰도**: 밴드폭이 좁을수록(스퀴즈 강할수록) 횡보 신뢰도 ↑

> ❓ **확인 필요 (3)**: 스퀴즈 임계값(밴드폭이 이 값보다 좁으면 횡보로 본다)은 종목마다 변동성이 달라서 절대값으로 못 박기 어려워. **"최근 120일 밴드폭의 하위 25% 구간이면 스퀴즈"** 같은 상대 기준으로 잡으려는데 괜찮아? (절대값 고정을 원하면 알려줘)

## 5. 타이밍 포착 엔진

추세 판별 결과에 따라 **다른 로직으로 분기**. 결과는 `매수`/`매도(또는 관망)`/`중립` + 타이밍 점수(%).

### 5.1 추세장 (상승/하락) — 일목 + MACD 중심, RSI 무시

| 신호 | 조건 | 점수 |
|---|---|---|
| 일목 | 주가가 구름대 위 유지 (상승 시 롱만, 하락 베팅 금지) | +가중 |
| MACD | 히스토그램 0선 위에서 확대 **또는** 골든크로스 발생 | +가중 |
| 볼린저 | 밴드워킹(주가가 상단 밴드를 타고 상승, %B ≥ 0.8 지속) | +가중 |

```
추세장_매수신호 =
    (주가 > 구름대상단)        × w_ichimoku
  + (MACD히스토 > 0 & 확대중)  × w_macd
  + (밴드워킹 감지)            × w_bb
타이밍점수 = (충족 가중합 / 최대 가중합) × 100
```
- 상승추세에서는 **매수/홀딩 신호만** 제시(하락 베팅 금지 원칙 반영)
- 하락추세에서는 매수 신호를 띄우지 않고 "관망/회피" 신호 위주

### 5.2 횡보장 — 볼린저 + RSI 중심, 일목/MACD 무시

| 신호 | 매수 타깃 | 매도 타깃 |
|---|---|---|
| 볼린저 | 밴드 하단 터치(%B ≤ 0.05) | 밴드 상단 터치(%B ≥ 0.95) |
| RSI | RSI ≤ 30 (과매도) | RSI ≥ 70 (과매수) |

```
횡보_고확률매수 = (밴드 하단 터치) AND (RSI ≤ 30)   # 둘 다 충족 시 최고점
횡보_고확률매도 = (밴드 상단 터치) AND (RSI ≥ 70)
타이밍점수 = 두 조건 충족 정도의 가중합 → %
```

### 5.3 가중치 초안 (config에서 조정)
| 추세장 | 값 | 횡보장 | 값 |
|---|---|---|---|
| w_ichimoku | 0.45 | w_bollinger | 0.55 |
| w_macd | 0.40 | w_rsi | 0.45 |
| w_bb(밴드워킹) | 0.15 | — | — |

## 6. 종합 평가 엔진

### 6.1 합산 공식 (확정된 가중치)
```
종합점수(0~100) =
    추세 기여   × 0.40
  + 타이밍 기여 × 0.40
  + 지표 보정   × 0.20
```

- **추세 기여**: 상승추세면 (신뢰도) 그대로, 하락추세면 (100 - 신뢰도)로 반전, 횡보면 50 기준
- **타이밍 기여**: 타이밍 점수(매수 방향이면 +, 매도/회피 방향이면 -로 환산)
- **지표 보정(20%)**: RSI 극단치, MACD 0선 위치, %B 위치 등 미세 가점/감점

### 6.2 5등급 매핑 (경계값은 config)
| 종합점수 | 등급 |
|---|---|
| 80 ~ 100 | 강한 추천 |
| 65 ~ 79 | 추천 |
| 40 ~ 64 | 보류 |
| 25 ~ 39 | 비추천 |
| 0 ~ 24 | 강한 비추천 |

### 6.3 가격대 역산 (휴리스틱 근사)
> 가격이 바뀌면 RSI·%B도 같이 바뀌므로 정밀 역산은 무겁다. 휴리스틱으로 **"이 가격 아래로 오면 매력 구간"** 식 근사를 제시.

- **'추천' 이상이 되는 가격대 (하단 매력 구간)**:
  - 횡보장: 볼린저 하단 부근 가격 + RSI 30 도달 예상가
  - 상승추세: 기준선(또는 구름 상단) 지지 가격 부근
- **'비추천' 이하가 되는 가격대 (과열/이탈 구간)**:
  - 볼린저 상단 돌파 + RSI 70 초과 예상가 (과열)
  - 또는 구름대 하향 이탈가 (추세 붕괴)

화면 표기 예: *"현재 73,000원 · 추천 전환 예상가 ≈ 68,500원 · 과열 경계 ≈ 81,000원"*

> ❓ **확인 필요 (4)**: 위 경계값(80/65/40/25)과 종합 가중치(40/40/20)는 초안이야. **일단 이대로 출발하고 나중에 조정**하는 방향 맞지? 처음부터 다른 값을 원하면 알려줘.

## 9. API 설계 (백엔드 ↔ 프론트)

| Method | Endpoint | 설명 | 응답 핵심 |
|---|---|---|---|
| GET | `/api/stock/{ticker}` | 종목 종합 분석 1회 호출 | 아래 통합 JSON |
| GET | `/api/stock/{ticker}/ohlc` | 차트용 봉 데이터 | 캔들 + 지표 시계열 |

**`/api/stock/{ticker}` 응답 구조(예시)**
```json
{
  "ticker": "005930.KS",
  "name": "삼성전자",
  "price": 73000,
  "change_pct": -1.2,
  "detail": { "market_cap": ..., "per": ..., "pbr": ...,
              "dividend_yield": ..., "volume": ...,
              "week52_high": ..., "week52_low": ... },
  "indicators": { "rsi": 41.2, "macd": {...},
                  "ichimoku": {...}, "bollinger": {...} },
  "trend": { "label": "횡보", "confidence": 62 },
  "timing": { "action": "매수 관심", "score": 71 },
  "evaluation": { "grade": "추천", "score": 68,
                  "buy_zone": 68500, "overheat_zone": 81000 }
}
```
> 프론트는 **분석 로직을 절대 갖지 않는다**. 모든 계산은 백엔드. 프론트는 받은 값을 그리기만 함(로직 일원화 → 버그·불일치 방지).

## 10. UI / UX 디자인 명세 (토스증권 스타일)

### 10.1 디자인 원칙
1. **여백이 주인공** — 요소를 빽빽하게 채우지 말고 넉넉한 패딩/간격. 한 화면에 정보 욕심 X.
2. **카드 기반 레이아웃** — 정보 묶음을 둥근 모서리 카드로 분리. 테두리 대신 아주 옅은 그림자.
3. **한 가지 강조색** — 포인트 컬러는 토스 블루 계열 하나만. 나머지는 무채색 + 의미색(등락).
4. **타이포 위계 명확** — 큰 숫자(가격)는 굵게/크게, 라벨은 작고 회색.
5. **선이 적은 화면** — 구분선 최소화, 색·간격으로 구역 구분.

### 10.2 컬러 토큰
| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg` | `#FFFFFF` / 다크 `#17171C` | 배경 |
| `--surface` | `#F9FAFB` | 카드 배경 |
| `--primary` | `#3182F6` (토스 블루) | 버튼·강조·링크 |
| `--text-strong` | `#191F28` | 제목·숫자 |
| `--text-weak` | `#8B95A1` | 라벨·보조 |
| `--up` | `#F04452` (빨강) | **상승/매수** (한국식) |
| `--down` | `#3182F6` (파랑) | **하락/매도** (한국식) |
| `--line` | `#F2F4F6` | 옅은 구분선 |

> ⚠️ **색상 관례**: 한국식(상승=빨강, 하락=파랑)으로 통일. 미국 종목도 동일 적용. 추후 설정에서 미국식(상승=초록) 토글을 옵션으로 둘 수 있음.

### 10.3 타이포그래피
- 폰트: **Pretendard** (한글·숫자 모두 깔끔, 토스 느낌). 영문 보조 시스템 폰트 폴백.
- 위계: 가격 28~32px Bold / 섹션 제목 18px SemiBold / 본문 15px / 라벨 13px (text-weak)

### 10.4 등급·추세 시각 표현
- **종합 등급 배지**: 5등급을 색+텍스트 칩으로.
  - 강한 추천(진한 빨강) · 추천(빨강) · 보류(회색) · 비추천(파랑) · 강한 비추천(진한 파랑)
- **신뢰도/타이밍 %**: 가는 막대 게이지(프로그레스 바)로 직관 표현. 막대 색 = 의미색.
- **추세 라벨**: 상승↗(빨강) / 하락↘(파랑) / 횡보→(회색) 화살표 아이콘 + 텍스트.

### 10.5 화면 구성

**① 종목 상세 화면 (세로 스크롤, 모바일 우선)**
```
┌──────────────────────────────┐
│ [검색바]                  ★   │  ← 상단 고정
├──────────────────────────────┤
│ 삼성전자  005930             │
│ 73,000원   ▼ -1.2%           │  ← 큰 숫자, 의미색
├──────────────────────────────┤
│ ┌── 종합 평가 카드 ──────┐    │
│ │  [추천]  종합 68점      │    │  ← 가장 눈에 띄게 최상단
│ │  추세: 횡보(신뢰 62%)   │    │
│ │  타이밍: 매수 관심 71%  │    │
│ │  추천 전환가 ≈ 68,500   │    │
│ │  과열 경계 ≈ 81,000     │    │
│ └────────────────────────┘    │
│ ┌── 본차트 카드 ─────────┐    │
│ │ 캔들+일목구름+볼린저    │    │
│ └────────────────────────┘    │
│ ┌── 보조차트 카드 ───────┐    │
│ │ RSI / MACD              │    │
│ └────────────────────────┘    │
│ ┌── 상세정보 카드 ───────┐    │
│ │ 시총·PER·PBR·배당...    │    │
│ └────────────────────────┘    │
└──────────────────────────────┘
```
- **종합 평가 카드를 최상단**에 둬서 "사도 되나?"라는 핵심 질문에 즉답.
- 차트는 그 아래. 깊게 보고 싶은 사람만 스크롤.

**② 즐겨찾기 대시보드**
```
┌──────────────────────────────┐
│ 관심 종목            정렬 ▾   │
├──────────────────────────────┤
│ 삼성전자  73,000 ▼1.2% [추천] │
│ NVDA     $..    ▲2.1% [강추] │

└──────────────────────────────┘
```
- 각 행 탭 → 상세 화면 이동. 종합등급 칩으로 한눈에 우선순위 파악.

### 10.6 마이크로 인터랙션 (토스 느낌의 핵심)
- 카드 등장: 부드러운 페이드+슬라이드(짧게, 과하지 않게)
- 숫자 변경: 살짝 카운트업
- 버튼/탭: 눌렀을 때 미세 축소(scale 0.98) 피드백
- 로딩: 스피너 대신 **스켈레톤 UI**(카드 모양 회색 깜빡임) → 고급스러움

### 10.7 면책 문구 (필수)
하단 또는 평가 카드에 작게:
> "본 분석은 기술적 지표의 기계적 계산 결과이며, 투자 권유나 수익 보장이 아닙니다. 투자 판단과 책임은 본인에게 있습니다."

## 11. 권장 폴더 구조
```
project/
├─ backend/
│  ├─ main.py            # FastAPI 진입점
│  ├─ config.py          # 모든 임계값/가중치 ★
│  ├─ data/loader.py     # yfinance + 캐시
│  ├─ indicators/        # rsi, macd, ichimoku, bollinger
│  ├─ engine/
│  │   ├─ trend.py       # 추세 판별
│  │   ├─ timing.py      # 타이밍 포착
│  │   └─ evaluation.py  # 종합 평가 + 가격역산
│  └─ api/routes.py
└─ frontend/
   ├─ src/
   │  ├─ components/     # Card, Badge, Gauge, Chart...
   │  ├─ pages/          # Detail, Dashboard
   │  ├─ api/            # 백엔드 호출
   │  ├─ store/          # 즐겨찾기(localStorage)
   │  └─ styles/tokens.css  # 디자인 토큰 ★
   └─ ...
```
- 차트 라이브러리 추천: **lightweight-charts**(TradingView 제작, 가볍고 금융차트 특화) 또는 **ECharts**.

> ❓ **확인 필요 (5)**: 차트 라이브러리 `lightweight-charts`로 갈까? 일목 구름대처럼 영역 칠하기는 ECharts가 더 유연한 면도 있어. 선호 없으면 lightweight-charts로 시작할게.

## 12. 남은 확인 사항 정리
개발 착수 전 답이 필요한 것들:
1. 백엔드 프레임워크 FastAPI 확정?
2. 캐시 TTL 10분 OK?
3. 스퀴즈 임계값 = "최근 120일 밴드폭 하위 25%" 상대기준 OK?
4. 종합 등급 경계(80/65/40/25)·가중치(40/40/20) 초안으로 출발 OK?
5. 차트 라이브러리 lightweight-charts OK?

> 위 5개만 답해주면 이 설계도는 **착수 가능한 완성본**이 돼. 그 다음 단계로는 각 모듈의 함수 시그니처·실제 코드 골격까지 내려갈 수 있어.
