# 투자 판단 앱 — 전문가급 설계도 (v1.1 · 구현 반영판)

> **문서 성격**: 본 설계도는 개발 시 옆에 두고 참조하는 기술 명세서다. 모든 수치·공식·임계값은 "근거 있는 초안"이며, 실제 운영 중 백테스트로 조정 가능하도록 **설정 파일(config)** 한 곳에 모은다.
>
> **이 판(v1.1)에 대해**: v1.0 설계 이후 실제로 구현된 앱에 맞춰 동기화한 문서다. v1.0의 미결정 항목("확인 필요 1~5")은 모두 확정되었고, 일부 설계는 구현 과정에서 더 정교해졌다(특히 §6.3 가격 역산, §9 API, §10 화면 구성). 구현과 어긋났던 부분은 구현 기준으로 정정했다.
>
> **확정된 전제**
> - 대상: 한국 + 미국 주식, 티커 직접 입력
> - 데이터: `yfinance` (비공식, 무료) / 일봉 기준 / 검색 시 스냅샷 1회 로드
> - yfinance 실패·차단 시 **결정론적 합성 데이터로 폴백**(앱이 죽지 않게), `source` 필드로 구분
> - 신뢰도·타이밍 %는 **통계적 확률이 아닌 규칙 기반 "신호 일치도(휴리스틱 점수)"**
> - 가격대 역산은 휴리스틱(근사) 방식
> - 즐겨찾기는 브라우저 localStorage 저장, 계정 시스템 없음
> - 화면: **상세 + 즐겨찾기 비교를 한 화면에 통합한 데스크톱 단일 대시보드**
> - UI: 토스증권 스타일(세련·깔끔·전문적)
> - 프론트엔드: **바닐라 JS + 네이티브 Canvas 렌더링**(빌드 도구·프레임워크 없음), 백엔드가 정적 서빙

## 2. 데이터 레이어  (`backend/data.py`)

### 2.1 데이터 수집
- 라이브러리: `yfinance`
- 호출: `yf.Ticker(ticker).history(period="1y", interval="1d", auto_adjust=False)`
- 지표 계산을 위해 **최소 1년치 일봉** 확보 (일목 선행스팬2가 52일 + 26일 시프트를 쓰므로 워밍업 구간 필요. 안전하게 1년)
- 봉이 60개 미만이거나 비어 있으면 해당 후보는 실패로 간주

### 2.2 티커 처리 규칙
| 시장 | 입력 예 | 내부 변환 |
|---|---|---|
| 미국 | `AAPL` | 그대로(대문자화) |
| 한국 6자리 | `005930` / `247540` | `.KS` → `.KQ` 순차 시도 |

> 숫자 6자리면 한국 종목으로 간주하고 `.KS` 먼저, 실패 시 `.KQ`를 시도한다. 전 후보가 실패하면 합성 폴백(§2.4)으로 넘어가되 첫 후보 심볼명을 유지한다.

### 2.3 캐시 (호출 제한 대응)
- yfinance는 과도 호출 시 일시 차단될 수 있음 → **동일 티커 N분 내 재요청은 메모리 캐시 반환**
- 메모리 캐시(서버 재시작 시 소멸). **TTL 10분** (`CACHE_TTL = 600`).
- 캐시에는 **수집 시각(`fetched_at`)** 을 함께 저장한다 → 캐시 히트로 응답해도 "실제로 데이터를 받아온 시각"이 보존된다(§2.5).

### 2.4 합성 데이터 폴백 (가용성 보장)
- yfinance가 차단/오프라인이어서 전 후보가 실패하면, 앱이 죽지 않도록 **결정론적 합성 OHLC**로 폴백한다.
- 티커 문자열 해시를 시드로 한 난수 보행(252봉) → 동일 티커는 항상 동일한 합성 데이터.
- 폴백 여부는 `source` 필드로 표시: `"live"`(실데이터) / `"synthetic"`(합성). 프론트는 상단 연결 상태를 `실시간 연결됨` ↔ `데모 데이터`로 바꿔 알린다.

### 2.5 갱신 시각 표기
- 일봉 데이터라 캔들 자체는 '일' 단위지만, 사용자가 "지금 보는 지표·그래프가 언제 받아온 것인지"를 알 수 있어야 한다.
- 응답에 두 가지를 담는다:
  - `updated_at`: 데이터를 **수집한 시각** `"YYYY-MM-DD HH:MM"` (캐시 보존된 `fetched_at` 기준 → 데이터 신선도)
  - `last_bar`: 데이터의 **최종 봉 날짜** `"YYYY-MM-DD"`
- 프론트 상단 연결 상태 옆에 `… HH:MM 갱신`으로 노출하고, 툴팁에 최종 봉 날짜를 보인다.

## 3. 지표 계산 (정확한 공식)  (`backend/indicators.py`)

> 모든 지표는 **일봉 종가/고가/저가** 기준. 파라미터는 표준값 사용, config로 변경 가능.
> 선행스팬은 **미시프트(원본 인덱스 정렬)** 상태로 계산해 두고, 시프트(26)는 표시·판정 단계에서 적용한다.

### 3.1 RSI (기간 14, Wilder 방식)
```
변화량 = 종가[t] - 종가[t-1]
상승폭 = max(변화량, 0),  하락폭 = max(-변화량, 0)
평균상승 = Wilder평활(상승폭, 14)
평균하락 = Wilder평활(하락폭, 14)
RS = 평균상승 / 평균하락
RSI = 100 - (100 / (1 + RS))
```
- Wilder 평활은 `ewm(alpha=1/14, adjust=False, min_periods=14)` 로 구현(첫 유효값까지 NaN).
- 과매수 70 / 과매도 30 (config)

### 3.2 MACD (12, 26, 9)
```
MACD선   = EMA(종가, 12) - EMA(종가, 26)
시그널선 = EMA(MACD선, 9)
히스토그램(오실레이터) = MACD선 - 시그널선
```
- 골든크로스: MACD선이 시그널선을 아래→위로 돌파 / 데드크로스: 그 반대

### 3.3 일목균형표 (9, 26, 52)
```
전환선     = (최근 9일 최고가 + 최근 9일 최저가) / 2
기준선     = (최근 26일 최고가 + 최근 26일 최저가) / 2
선행스팬1  = (전환선 + 기준선) / 2          → 표시 시 26일 앞으로 이동
선행스팬2  = (최근 52일 최고가 + 최근 52일 최저가) / 2  → 표시 시 26일 앞으로 이동
후행스팬   = 당일 종가                       → 26일 뒤로 이동
구름대(雲) = 선행스팬1 ~ 선행스팬2 사이 영역
```
- 양운(상승 구름): 선행스팬1 > 선행스팬2 / 음운(하락 구름): 선행스팬1 < 선행스팬2

### 3.4 볼린저밴드 (20, 2σ)
```
중심선 = SMA(종가, 20)
상단   = 중심선 + 2 × 표준편차(종가, 20)      # 모표준편차(ddof=0)
하단   = 중심선 - 2 × 표준편차(종가, 20)
%B     = (현재가 - 하단) / (상단 - 하단)        # 0=하단, 1=상단
밴드폭 = (상단 - 하단) / 중심선                  # 변동성/스퀴즈 판단
```

### 3.5 차트 배치
- **본차트(메인)**: 캔들 + 일목균형표(전환·기준·구름·후행) + 볼린저밴드(상·중·하). 선행스팬은 26일 앞으로 시프트되어 **차트 우측에 미래 구름대**로 그려진다.
- **보조차트**: RSI(0~100, 30/70 가이드라인), MACD(MACD선·시그널선·히스토그램 막대)

## 4. 추세 판별 엔진  (`engine.analyze_trend`)

일목균형표 + 볼린저밴드로 **기계적 판정**. 결과는 `상승추세` / `하락추세` / `횡보(박스권)` + 신뢰도(%).

### 4.1 판정에 쓰는 조건 (각 조건 = 1점)

| # | 조건 | 상승 신호 | 하락 신호 |
|---|---|---|---|
| C1 | 주가 vs 구름대 | 구름대 위 | 구름대 아래 |
| C2 | 구름 색 | 양운 | 음운 |
| C3 | 전환선 vs 기준선 | 전환선 > 기준선 | 전환선 < 기준선 |
| C4 | 후행스팬(당일 종가) vs 26일 전 주가 | 위 | 아래 |
| C5 | 볼린저 중심선 기울기 | 상승 | 하락 |

> 현재 봉에 걸리는 구름대는 "26일 전에 계산된 선행스팬"을 사용한다(일목 정의 그대로). 26일 전 값이 없으면 당일 값으로 폴백.

### 4.2 판정 로직
```
상승점수 = (상승 신호인 조건 개수)   # 0~5
하락점수 = (하락 신호인 조건 개수)   # 0~5

if 주가가 구름대 내부 또는 스퀴즈:
    추세 = "횡보"
elif 상승점수 >= 3 and 상승점수 > 하락점수:
    추세 = "상승추세"
elif 하락점수 >= 3 and 하락점수 > 상승점수:
    추세 = "하락추세"
else:
    추세 = "횡보"   # 신호 충돌 시 보수적으로 횡보 처리
```
- 추세 인정 최소 일치 조건 수 = 3 (`TREND_MIN_SCORE`)

### 4.3 신뢰도(%) 산출
```
상승추세 신뢰도 = (상승점수 / 5) × 100
하락추세 신뢰도 = (하락점수 / 5) × 100
횡보   신뢰도 = (1 - rank) × 100
   rank = 최근 120일 밴드폭 중 현재 밴드폭보다 좁은 비율  # 0=가장 좁음
```
- **횡보 신뢰도**: 밴드폭이 좁을수록(스퀴즈 강할수록) 횡보 신뢰도 ↑

### 4.4 스퀴즈 판정
- **"최근 120일 밴드폭의 하위 25% 이내이면 스퀴즈"** 라는 상대 기준(`SQUEEZE_LOOKBACK=120`, `SQUEEZE_PCTL=0.25`). 종목마다 변동성이 달라 절대값 고정 대신 상대 분위수를 쓴다.

## 5. 타이밍 포착 엔진  (`engine.analyze_timing`)

추세 판별 결과에 따라 **다른 로직으로 분기**. 결과는 `action`(매수/홀딩/관망/매수 관심/매도 관심/중립) + 타이밍 점수(%) + 내부 `direction`(부호: 매수+ / 회피−).

### 5.1 추세장 (상승/하락) — 일목 + MACD 중심, RSI 무시
```
ich      = 1 if 주가 > 구름대상단 else 0
macd_sig = 1 if (히스토 > 0 and 히스토 ≥ 전일 히스토) else 0
walk     = 1 if (%B ≥ 0.8 and 전일 %B ≥ 0.8) else 0   # 밴드워킹
타이밍점수 = (ich·0.45 + macd_sig·0.40 + walk·0.15) × 100
```
- **상승추세**: `매수`(점수 ≥ 70) / `홀딩`(≥ 45) / `관망`(그 외). direction = +1.0 / +0.3 / −0.2
- **하락추세**: 매수 신호 금지 → 항상 `관망`, direction = −1.0 (하락 베팅도 제시하지 않음)

### 5.2 횡보장 — 볼린저 + RSI 중심, 일목/MACD 무시 (그라데이션 점수)
```
buy  = (0.55·clamp((0.30 − %B)/0.30) + 0.45·clamp((50 − RSI)/40)) × 100
sell = (0.55·clamp((%B − 0.70)/0.30) + 0.45·clamp((RSI − 50)/40)) × 100

if buy ≥ sell and buy ≥ 40 :  action="매수 관심", 점수=buy,  dir=+1
elif sell > buy and sell ≥ 40: action="매도 관심", 점수=sell, dir=−1
else:                         action="중립",     점수=max(buy,sell), dir=0
```
> v1.0의 "밴드 하단 터치(%B≤0.05) AND RSI≤30" 이진 조건 대신, 하단/과매도에 **가까울수록 점수가 연속적으로 오르는** 그라데이션으로 구현했다. 막대 게이지로 보여주기에도 자연스럽다.

### 5.3 가중치 (config에서 조정)
| 추세장 | 값 | 횡보장 | 값 |
|---|---|---|---|
| w_ichimoku | 0.45 | w_bollinger | 0.55 |
| w_macd | 0.40 | w_rsi | 0.45 |
| w_bb(밴드워킹) | 0.15 | — | — |

## 6. 종합 평가 엔진  (`engine.evaluate` / `_score_core`)

### 6.1 합산 공식 (확정된 가중치)
```
종합점수(0~100) =
    추세 기여   × 0.40
  + 타이밍 기여 × 0.40
  + 지표 보정   × 0.20
```

- **추세 기여**: 상승추세면 (신뢰도) 그대로, 하락추세면 (100 − 신뢰도)로 반전, 횡보면 50 기준
- **타이밍 기여** (0~100, 50=중립):
  - 일반: `clamp(50 + direction × (타이밍점수 / 2))`
  - **하락추세 보정**: 하락추세의 "관망"은 매수 타이밍 점수가 ≈0이라 위 식이 50(중립)에 묶여 점수 하한이 생긴다. 그래서 하락추세에서는 `50 − (하락 신뢰도)/2` 로 **하방 신뢰도에 비례해 분명히 낮춘다**.
- **지표 보정(20%)**: 50 기준에서 미세 가감 → `(50−RSI)/50×20` + `(히스토>0 ? +5 : −5)` + `(MACD선>0 ? +5 : −5)` + `(0.5−%B)×10`, 0~100 클램프

### 6.2 5등급 매핑 (경계값은 config)
| 종합점수 | 등급 |
|---|---|
| 80 ~ 100 | 강한 추천 |
| 65 ~ 79 | 추천 |
| 40 ~ 64 | 보류 |
| 25 ~ 39 | 비추천 |
| 0 ~ 24 | 강한 비추천 |

### 6.3 등급 전환 예상가 (전체 파이프라인 역산)
> v1.0의 단순 "추천 전환가 / 과열 경계" 휴리스틱을 **실측 역산**으로 강화했다. "가격이 이 수준에 닿으면 종합 **등급**이 한 단계 바뀐다"를 실제로 계산한다.

- 동작(`_scan_transition` + `_pipeline_score`):
  1. 현재가에서 위(+)·아래(−) 방향으로 **1.5% 격자**, 최대 **±40%** 까지 후보가 P를 훑는다.
  2. 각 P에 대해 마지막 봉 종가를 P로 가정(필요 시 고가/저가 보정)하고 **전 지표·추세·타이밍·종합점수를 재계산**해 등급을 구한다.
  3. 등급이 처음으로 한 단계 넘어가는 구간을 찾고, **4회 이분탐색**으로 전환가를 좁힌다.
  4. 한 칸 더 갔을 때 되돌아오는 **국소 노이즈는 건너뛴다**.
- 출력: `up_price`/`up_grade`(오르면 한 단계 ↑ 되는 가격·등급), `down_price`/`down_grade`(내리면 한 단계 ↓ 되는 가격·등급).
- ±40% 안에 전환이 없으면 `None`. 현재 등급이 최고(강한 추천)면 상향이, 최저(강한 비추천)면 하향이 자연히 `None` → 프론트는 "현재 최고/최저 등급"으로 표기.

> 화면 표기 예: *"상향 전환 → 추천  ≈ 81,000원 · 하향 전환 → 비추천  ≈ 68,500원"*

## 9. API 설계 (백엔드 ↔ 프론트)  (`backend/main.py`)

| Method | Endpoint | 설명 | 응답 핵심 |
|---|---|---|---|
| GET | `/api/stock/{ticker}` | 종목 종합 분석 1회 호출 | 아래 통합 JSON |
| GET | `/api/stock/{ticker}/ohlc` | 차트용 봉 + 지표 시계열 | 캔들 + 지표 배열 |
| GET | `/api/health` | 헬스체크 | `{ "ok": true }` |
| GET | `/`, `/*` | 프론트 정적 파일 | `index.html` 등 |

**`/api/stock/{ticker}` 응답 구조(예시)**

  "updated_at": "2026-06-20 18:18",
  "last_bar": "2026-06-18",
  "detail": { "market_cap": "...", "sector": "...", "per": "...", "pbr": "...",
              "dividend_yield": "...", "volume": "...",
              "week52_high": "...", "week52_low": "..." },
  "indicators": { "rsi": 41.2 },
  "trend": { "label": "횡보", "confidence": 62,
             "up_score": 2, "down_score": 1, "squeeze": false },

## 10. UI / UX 디자인 명세 (토스증권 스타일)

### 10.1 디자인 원칙
1. **여백이 주인공** — 넉넉한 패딩/간격. 한 화면에 정보 욕심 X.
2. **카드 기반 레이아웃** — 정보 묶음을 둥근 모서리 카드로 분리. 테두리 대신 아주 옅은 그림자.
3. **한 가지 강조색** — 포인트 컬러는 토스 블루 계열 하나만. 나머지는 무채색 + 의미색(등락).
4. **타이포 위계 명확** — 큰 숫자(가격)는 굵게/크게, 라벨은 작고 회색.
5. **선이 적은 화면** — 구분선 최소화, 색·간격으로 구역 구분.

### 10.2 컬러 토큰
| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg` | `#E4E8ED` / `#F4F6F8` | 배경 |
| `--surface` | `#FFFFFF` | 카드 배경 |
| `--primary` | `#3182F6` (토스 블루) | 버튼·강조·링크 |
| `--text-strong` | `#191F28` | 제목·숫자 |
| `--text-weak` | `#8B95A1` | 라벨·보조 |
| `--up` | `#F04452` (빨강) | **상승/매수** (한국식) |
| `--down` | `#3182F6` (파랑) | **하락/매도** (한국식) |
| `--line` | `#F2F4F6` | 옅은 구분선 |

> ⚠️ **색상 관례**: 한국식(상승=빨강, 하락=파랑)으로 통일. 미국 종목도 동일 적용.

### 10.3 타이포그래피
- 폰트: **Pretendard** (CDN). 영문 보조 시스템 폰트 폴백.
- 위계: 가격 38px Bold / 종목명 27px Bold / 섹션 제목 14~16px SemiBold / 본문 13~14px / 라벨 11~12px (text-weak)

### 10.4 등급·추세·신호 시각 표현
- **종합 등급 배지**: 5등급 색+텍스트 칩 — 강한 추천(진한 빨강)·추천(빨강)·보류(회색)·비추천(파랑)·강한 비추천(진한 파랑). 사이드바 칩은 약칭(강추/추천/보류/비추/강비).
- **종합 점수**: 108px 도넛 게이지 + 가운데 점수/100, 등급색으로 채움. 카운트업 애니메이션.
- **신뢰도/타이밍 %**: 가는 막대 게이지(프로그레스 바), 막대 색 = 의미색.
- **추세 라벨**: 상승↗(빨강) / 하락↘(파랑) / 횡보→(회색) 화살표 + 텍스트.
- **지표 신호 요약**: 일목·RSI·MACD·볼린저·이동평균(20) 각각을 매수/매도/중립 칩으로, 합산 가중치로 `매수 우위 / 매도 우위 / 혼조` 컨센서스 표기.

### 10.5 화면 구성 — 데스크톱 단일 대시보드 (고정 1440×920)

상세 화면과 즐겨찾기 비교를 **한 화면에 통합**한 3분할 레이아웃이다.

## 11. 폴더 구조 (실제 구현)
```
투자 지표 자동화기/
├─ backend/
│  ├─ main.py          # FastAPI 진입점 + 라우트 + 프론트 정적 서빙

│  └─ engine.py        # 추세 판별 · 타이밍 · 종합 평가 · 가격 역산 · 신호 요약
├─ frontend/
│  ├─ index.html       # 단일 화면(인라인 스타일)
│  └─ app.js           # 네이티브 Canvas 렌더러 + /api 호출 + localStorage

└─ 설계도.md
```
- 차트: **원본 디자인의 네이티브 Canvas 렌더러**를 그대로 사용(lightweight-charts/ECharts 미도입). 캔들·일목 구름·볼린저·미래 구름까지 직접 그린다.
- 실행: `python -m venv .venv && pip install -r requirements.txt && uvicorn backend.main:app` → `http://127.0.0.1:8000` (또는 `launch.bat`).
